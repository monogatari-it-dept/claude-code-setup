# テストルール

- テストファイルはソースファイルと同階層に `.test.ts` で作成する
- テスト名は日本語で記述してよい（可読性を重視）
- Arrange-Act-Assert パターンに従う
- モックは最小限にする。実際のデータベースやAPIに依存する場合のみ使用
- テストデータはファクトリ関数で作成する（ハードコーディングしない）
- 各テストは他のテストに依存しない独立したものにする
- エッジケース（null, undefined, 空配列, 境界値）を必ずテストする
- 非同期処理のテストにはasync/awaitを使用する
- カバレッジ目標: ビジネスロジック80%以上、ユーティリティ関数90%以上
